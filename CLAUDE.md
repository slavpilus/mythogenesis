# CLAUDE.md

Guidance for Claude Code when working in this repository.

## What this is

Mythogenesis: a Gradio Space for the Hugging Face Build Small Hackathon (creative
track). A semantic, narrated Conway's Game of Life where live cells carry words and
colors, novel children are generated by an LLM from parent words, and a narrator
writes rolling prose. Living symbols reproduce under finite rules until a few become myth.

**`handover.md` is the source of truth for the spec.** It contains the full design,
state model, step algorithm, prompts, config values, required tests, build order, and
definition of done. Read the relevant section before implementing; section numbers are
referenced throughout the code stubs.

## Environment & commands

This project uses **uv** (not pip/venv). Always use uv:

```bash
uv sync                       # create env + install deps (runtime + dev)
uv run python app.py          # launch the Gradio app locally
uv run pytest                 # run tests
uv run pytest --cov           # tests with coverage
uv run ruff check .           # lint
uv run ruff format .          # format
uv add <pkg>                  # add a runtime dependency
uv add --dev <pkg>            # add a dev dependency
```

Python is pinned to 3.11 (`.python-version`) for Gradio / HF Spaces compatibility.

## Architecture

Keep all game logic out of `app.py` and out of Gradio so the core stays unit-testable
(tests must not require Gradio or network access). Module responsibilities (handover §13):

- `config.py` - the single config block (board size, thresholds, TTLs, demo seed). Filled.
- `life_core.py` - Conway mechanics only: toroidal neighbor counting, B3/S23 step.
- `semantics.py` - word registry, duplicate-safe parent signatures, progeny/promotion,
  myth pool with TTL/cap, rolling story trim, strongest-neighbor inheritance.
- `model_backend.py` - the three public functions (`generate_births`, `narrate_step`,
  `write_epitaph`) behind which all HF Inference API code lives, plus a deterministic
  local fallback so the app runs with no token and never crashes on bad model output.
- `render.py` - pure SVG/HTML string builders for the sky / grid / story bands.
- `seed_patterns.py` - seed text cleaning + deterministic placement of curated moving
  patterns (glider, toad, beacon, LWSS) on collision courses.
- `app.py` - Gradio wiring only (single `gr.Blocks`, three bands, controls).

## Conventions

- **uv for all Python work.** Never invoke pip/venv directly.
- **No AI attribution in git commits or PR bodies.** Plain messages, no co-author trailer.
- **No em dashes** in generated docs or in model prose (hackathon judging rule, handover §17).
- **TDD for the core.** `tests/test_life_core.py`, `test_semantics.py`, `test_story.py`
  hold the handover §12 checklist as skipped stubs. Implement each test, make it pass,
  remove its skip mark. Tests must not require Gradio or the network.
- Keep core logic deterministic and Gradio-free so it is testable.
- Model output must be parsed defensively (strip code fences, validate JSON / hex / single
  lowercase word) and fall back to inheritance rather than crashing the loop.

## CI/CD & deployment

- `.github/workflows/ci.yml` - runs on push to main, PRs, and manual dispatch:
  uv sync, ruff check, ruff format check, pytest+coverage. Keep it green.
- `.github/workflows/deploy.yml` - after CI succeeds on main, force-pushes the repo to
  the Hugging Face Space. Uses GitHub secret `HF_TOKEN` and variable `HF_SPACE_ID`.
  Exits cleanly (no deploy) if those are unset.
- The HF Space card is driven by the YAML frontmatter at the top of `README.md`
  (`sdk: gradio`, `sdk_version`, `app_file: app.py`, `python_version`). `requirements.txt`
  is the Space runtime install list; keep it in sync with pyproject runtime deps.
